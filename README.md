# dmytro.my

single-page site · cli aesthetic · plain html/css/js, with a tiny no-deps node build for blog posts (SEO).

## tree

```
.
├── index.html              home page
├── about/
│   └── index.html          about page (~/about)
├── build.js                pre-renders localized blog routes (run before pushing)
├── sitemap.xml             auto-generated by build.js
├── robots.txt              auto-generated by build.js
├── styles/main.css         site + blog styles
├── scripts/
│   ├── main.js             shared page interactions: nav, scramble, spotlight, clock
│   ├── markdown.js         minimal md renderer (used by build.js + listing)
│   ├── blog.js             hydrates localized listings from posts.json
│   └── post-static.js      runtime niceties on pre-rendered post pages
├── assets/favicon.svg
├── .nojekyll               disable Jekyll on GitHub Pages
└── blog/
    ├── index.html          English listing (`/blog/`)
    ├── post.html           legacy redirect (?slug= → posts/<slug>/)
    ├── posts.json          site config + post manifest (lang per post)
    ├── feed.xml            RSS — all languages
    ├── feed.en.xml         RSS — english only
    ├── feed.ru.xml         RSS — russian only
    └── posts/
        ├── <slug>.md       source post
        └── <route>/
            └── index.html  English pre-rendered post
```

Non-default languages are generated under locale-prefixed routes:

```text
ru/blog/index.html
ru/blog/posts/<route>/index.html
```

## run locally

`fetch()` won't work over `file://`, so the listing needs a local http server:

```bash
npx serve .
# or
python -m http.server 8000
```

Post pages are pre-rendered under the locale-aware public routes shown above.

## adding a post

1. Drop `blog/posts/<slug>.md` (h1 first line is the title).
2. Add an entry to the `posts` array in `blog/posts.json`:

```json
{
  "slug": "<unique-source-slug>",
  "route": "<shared-public-route>",
  "translationKey": "<shared-translation-key>",
  "title": "post title",
  "lang": "en",
  "date": "YYYY-MM-DD",
  "tags": ["tag1", "tag2"],
  "summary": "one-line teaser"
}
```

`lang` is required and must be one of `site.languages` (currently `en`, `ru`).
One manifest entry = one language. Translations use separate source slugs, the same `route`, and the same `translationKey`.

3. Run the build:

```bash
node build.js
```

This pre-renders the post into its locale-aware route (with `<title>`, `<meta description>`, OG tags, Twitter cards, JSON-LD `BlogPosting`, `<html lang>`, canonical and `hreflang` links), and regenerates localized listings, `sitemap.xml`, `blog/feed.xml`, and per-language feeds (`blog/feed.<lang>.xml`).

4. Commit everything (md sources, manifest, generated html, sitemap, feeds).

## languages

- Configured in `blog/posts.json` → `site.languages` and `site.defaultLang`.
- Currently: `en` (default), `ru`.
- Each locale has its own listing. EN/RU links navigate between `/blog/` and `/ru/blog/`; there is no mixed-language `all` mode.
- Each post page is fully localized at the `<head>` level (`<html lang>`, `og:locale`, JSON-LD `inLanguage`).
- English uses `/blog/`; non-default languages use `/<lang>/blog/`.
- Public routes stay English-lowercase even for non-English posts.
- Listings never mix languages. EN/RU links navigate between locale pages.
- Tags stay English-lowercase across all languages so the tag cloud doesn't fragment.

## SEO

- Each post is a real static HTML page at `/blog/posts/<route>/` or `/<lang>/blog/posts/<route>/`.
- Full `<title>`, `<meta description>`, Open Graph (with locale), Twitter Card, JSON-LD `BlogPosting` (with `inLanguage`) on every post.
- `<html lang>` is set per post (en/ru).
- `sitemap.xml`, `robots.txt`, RSS feeds: combined `feed.xml` and per-language `feed.<lang>.xml`.
- Listing, archive, and tags are pre-rendered per language, then hydrated from `posts.json` in the browser.
- Update `blog/posts.json` `site.url` if you change domains.

## deploy

GitHub Pages — point at the repo root. `.nojekyll` is already in place so Pages won't try to process markdown itself.

## security (public repo)

Before every commit/push, review what was created and what is being published:

```bash
git status
git diff --staged
```

Sensitive data (logins/passwords/tokens/keys/auth files) must never be committed.
See full rules in [SECURITY_RULES.md](./SECURITY_RULES.md).
