# dmytro.my

single-page site · cli aesthetic · plain html/css/js, with a tiny no-deps node build for blog posts (SEO).

## tree

```
.
├── index.html              entry
├── build.js                pre-renders blog posts → static html (run before pushing)
├── sitemap.xml             auto-generated by build.js
├── robots.txt              auto-generated by build.js
├── styles/main.css         site + blog styles
├── scripts/
│   ├── main.js             home: nav, scramble, spotlight, clock
│   ├── markdown.js         minimal md renderer (used by build.js + listing)
│   ├── blog.js             blog listing (client-side from posts.json)
│   └── post-static.js      runtime niceties on pre-rendered post pages
├── assets/favicon.svg
├── .nojekyll               disable Jekyll on GitHub Pages
└── blog/
    ├── index.html          listing (~/blog)
    ├── post.html           legacy redirect (?slug= → posts/<slug>/)
    ├── posts.json          site config + post manifest
    ├── feed.xml            RSS 2.0 (auto-generated)
    └── posts/
        ├── <slug>.md       source post
        └── <slug>/
            └── index.html  pre-rendered (auto-generated)
```

## run locally

`fetch()` won't work over `file://`, so the listing needs a local http server:

```bash
npx serve .
# or
python -m http.server 8000
```

Pre-rendered posts (`blog/posts/<slug>/index.html`) work fine via `file://` once built.

## adding a post

1. Drop `blog/posts/<slug>.md` (h1 first line is the title).
2. Add an entry to the `posts` array in `blog/posts.json`:

```json
{
  "slug": "<slug>",
  "title": "post title",
  "date": "YYYY-MM-DD",
  "tags": ["tag1", "tag2"],
  "summary": "one-line teaser"
}
```

3. Run the build:

```bash
node build.js
```

This pre-renders the post into `blog/posts/<slug>/index.html` (with `<title>`, `<meta description>`, OG tags, Twitter cards, JSON-LD `BlogPosting`, canonical URL), and regenerates `sitemap.xml` and `blog/feed.xml`.

4. Commit everything (md sources, manifest, generated html, sitemap, feed).

## SEO

- Each post is a real static html page at `/blog/posts/<slug>/`.
- Full `<title>`, `<meta description>`, Open Graph, Twitter Card, JSON-LD on every post.
- `sitemap.xml`, `robots.txt`, RSS feed at `/blog/feed.xml`.
- Listing/archive/tags still hydrate from `posts.json` via fetch (those pages aren't critical for SEO).
- Update `blog/posts.json` `site.url` if you change domains.

## deploy

GitHub Pages — point at the repo root. `.nojekyll` is already in place so Pages won't try to process markdown itself.

## security (public repo)

Before every commit/push, review what was created and what is being published:

```bash
git status
git diff --staged
```

Sensitive data (logins/passwords/tokens/keys/auth files) must never be committed.
See full rules in [SECURITY_RULES.md](./SECURITY_RULES.md).
