# OpenClaw: личный ИИ-ассистент для всех устройств

OpenClaw - это open-source-ассистент, который работает на вашей инфраструктуре и отвечает в привычных каналах связи.

В центре системы находится Gateway. Он соединяет модели, инструменты, память, устройства и мессенджеры в одну систему.

## один Gateway - много каналов

OpenClaw поддерживает WhatsApp, Telegram, Slack, Discord, Signal, iMessage, Microsoft Teams, Matrix и другие платформы.

Это не просто одинаковый бот в нескольких мессенджерах. Gateway управляет сессиями, маршрутизацией, инструментами и событиями.

Можно создать несколько изолированных агентов. Например:

- личного ассистента;
- агента для разработки;
- бота поддержки;
- агента для командного чата.

Для каждого задаётся собственная рабочая директория, история, память и набор разрешений.

## память

OpenClaw хранит долгосрочную память в Markdown-файлах внутри workspace. Также доступен встроенный SQLite-бэкенд и дополнительные системы памяти: QMD, Honcho и LanceDB.

Перед сжатием длинного диалога агент может сохранить важные факты в память. Это помогает не потерять информацию во время compaction.

Память остаётся частью рабочей директории, поэтому её можно читать, редактировать, резервировать и хранить в собственной инфраструктуре.

## skills и ClawHub

Skills в OpenClaw - это инструкции в формате `SKILL.md`, которые объясняют агенту, как использовать инструменты и выполнять конкретные процессы.

Они могут находиться на нескольких уровнях:

- внутри workspace;
- в проекте;
- в личном каталоге пользователя;
- в общем каталоге OpenClaw;
- внутри плагинов.

Приоритет локальных skills выше встроенных. Поэтому стандартное поведение можно переопределять без изменения исходного кода.

Готовые skills распространяются через ClawHub. Также есть Skill Workshop для просмотра и подтверждения процедур, подготовленных агентом.

## cron и heartbeat

В OpenClaw есть два разных механизма автоматизации.

Cron подходит для точных задач: отправить отчёт утром, запустить резервное копирование ночью или выполнить проверку в определённое время.

Heartbeat используется для периодической фоновой активности, когда точное время не так важно: проверить состояние сервисов, новые сообщения или незавершённые задачи.

Есть также фоновые tasks и webhooks. Поэтому OpenClaw можно использовать не только как реактивного бота, но и как постоянный автоматизированный процесс.

## безопасность

Основная сессия по умолчанию может выполнять инструменты на хосте. Это удобно, но требует осторожности.

Для дополнительных агентов и групповых каналов можно включить sandbox. OpenClaw поддерживает Docker, SSH и OpenShell как бэкенды изоляции.

Неизвестные пользователи в мессенджерах по умолчанию проходят pairing. Агент не начинает обрабатывать их сообщения, пока владелец не подтвердит код подключения.

Экспорт диагностики через OpenTelemetry подключается отдельно. Содержимое промптов, ответов и tool calls по умолчанию не экспортируется.

## кому подойдёт OpenClaw

OpenClaw стоит рассмотреть, если нужен:

- постоянно работающий личный ассистент;
- доступ через множество мессенджеров;
- iMessage;
- несколько изолированных агентов;
- единый Gateway для устройств и каналов;
- большой каталог готовых skills;
- автоматизация через cron, heartbeat и webhooks.

## что проверить перед запуском

Перед подключением реальных аккаунтов я бы:

- запустил Gateway локально;
- включил pairing для личных сообщений;
- вынес групповые сессии в sandbox;
- ограничил доступ к файлам и командам;
- создал отдельные API-ключи с минимальными правами;
- проверил резервное копирование workspace;
- подключал каналы по одному.

OpenClaw - это уже не просто бот для Telegram или WhatsApp. Это локальный control plane для личного ИИ-ассистента, который может работать на разных устройствах и общаться через десятки каналов.

Его сильная сторона - не одна конкретная модель, а инфраструктура вокруг агента: Gateway, маршрутизация, память, skills, автоматизация и безопасность.
